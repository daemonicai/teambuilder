## ADDED Requirements

### Requirement: Reviewer persona runs at archive time

`/opsx:archive` SHALL invoke the Reviewer persona in a fresh subagent before finalizing archival of a change. The Reviewer subagent MUST receive pointers to the change directory (`proposal.md`, `design.md`, `specs/`, `tasks.md`) and the code diff produced while implementing the change.

#### Scenario: User runs `/opsx:archive` on a completed change

- **WHEN** a user runs `/opsx:archive` on a change whose tasks are all complete
- **THEN** `/opsx:archive` invokes the Reviewer persona via the Agent tool before archiving
- **AND** the Reviewer subagent prompt contains paths to the change directory's artifacts and a diff of the code changes

#### Scenario: Reviewer persona file is missing

- **WHEN** a user runs `/opsx:archive` and the Reviewer persona file does not exist in `.claude/agents/`
- **THEN** `/opsx:archive` stops and informs the user to run `/teambuild:reviewer` before archiving

### Requirement: Reviewer findings are surfaced but non-blocking

The Reviewer persona SHALL return findings that `/opsx:archive` presents to the user. Findings MUST NOT block archival; the user SHALL decide whether to address findings first or proceed with archival.

#### Scenario: Reviewer returns findings

- **WHEN** the Reviewer subagent returns one or more findings
- **THEN** `/opsx:archive` displays the findings to the user
- **AND** it asks the user whether to proceed with archival or pause to address findings first
- **AND** it proceeds with archival if the user chooses to proceed

#### Scenario: Reviewer returns no findings

- **WHEN** the Reviewer subagent returns an empty findings list
- **THEN** `/opsx:archive` proceeds directly with archival without further prompting

### Requirement: Reviewer persona declares archive-time review as a fixed duty

The Reviewer persona file generated by `/teambuild:reviewer` SHALL declare "per-change review at archive time" as a named duty in its fixed core. The duty description MUST state that the review covers the full change directory and code diff and produces non-blocking findings.

#### Scenario: User inspects the generated Reviewer persona file

- **WHEN** a user opens `.claude/agents/reviewer.md` after running `/teambuild:reviewer`
- **THEN** the file contains a named duty identifying per-change review at archive time as part of the Reviewer's role
- **AND** the duty describes the review scope (full change directory and code diff) and its non-blocking nature
